# 开屏吉祥物动画实现计划

## 概述
为阿柑少年网站创建一个炫酷的开屏吉祥物动画，时长6-8秒，包含多个动画阶段。

## 技术栈
- React 18 + TypeScript
- Framer Motion（动画库）
- Tailwind CSS（样式）
- 现有吉祥物图像（mascot-full.png）

## 动画分镜脚本（根据用户要求）
1. **0.00 – 1.20 秒**：黑场/极暗背景，很轻的环境音
2. **1.20 – 2.20 秒**：橙色圆形慢慢浮现
3. **2.20 – 3.20 秒**：角色轮廓慢慢完整（耳朵/角、尾巴轻轻长出来）
4. **3.20 – 4.40 秒**：表情出现，吉祥物第一次"呼吸"
5. **4.40 – 5.50 秒**：周围出现极轻微自然元素
6. **5.50 – 6.80 秒**：文案淡入
7. **6.80 – 8.00 秒**：吉祥物看向观众/轻轻眨眼/进入idle状态

## 实现方案

### 1. 创建 SplashAnimation 组件
- 位置：`src/components/SplashAnimation.tsx`
- 功能：全屏覆盖，实现所有动画阶段
- 使用 Framer Motion 实现复杂的动画序列

### 2. 动画实现策略
- 使用 `useState` 和 `useEffect` 控制动画阶段
- 使用 Framer Motion 的 `motion` 组件实现动画效果
- 使用 CSS 遮罩和伪元素模拟吉祥物各部分的逐渐出现
- 使用 `requestAnimationFrame` 实现呼吸动画

### 3. 集成到现有网站
- 修改 `HeroMascotStage` 组件，在首页加载时显示开屏动画
- 动画结束后，吉祥物平滑过渡到现有位置
- 使用 `localStorage` 确保动画只在首页显示一次

### 4. 文案和自然元素
- 使用 Framer Motion 实现文案淡入效果
- 添加叶子、微光点等自然元素动画
- 实现眨眼和尾巴摆动动画

### 5. 性能优化
- 使用 `will-change` 属性优化动画性能
- 避免不必要的重绘和回流
- 使用 `AnimatePresence` 管理动画生命周期

## 注意事项
- 动画只在首页显示
- 动画结束后吉祥物作为常驻角色保持在现在的位置
- 不需要考虑无障碍设置（用户明确表示）
- 动画要非常炫酷，有设计感和交互感
